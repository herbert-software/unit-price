## 为什么

`我的` Tab 现为「敬请期待」占位页,不发任何请求。它要存在的真正理由有两个现成抓手:① 即时比价 `POST /compute` 已上线,但入口只埋在「搜索未命中兜底 CTA」里,几乎不可发现;② 比价是一次性、无状态的,用户算完即丢,没有"再看一眼上次"的去处。把这两件事放进 `我的`,Tab 就从占位变成有用,且**全部在端上本地完成、不破只读边界**。

## 变更内容

`我的` Tab 从占位页升级为两个区(均**不引入登录、不写库、不碰众包/抓取**):

- **比价工具区**
  - **即时比价常驻入口**:`navigateTo` 到已存在的比价表单页(`pages/compute`),让埋没的 `/compute` 被看见。`我的` 自身**仍不发起任何网络请求**(请求发生在 compute 页)。
  - **最近比价记录**:比价表单页在 `POST /compute` 成功后,把本次**结构化输入**写入端上 `Taro.setStorage`(环形保留最近 N 条);`我的` 读取并列出,点一条回填重算。纯本地草稿,**非**录入到库、**非**贡献/纠错。
- **关于区**
  - **数据来源与时效说明**:静态文案——众包贡献 + 运营整理校准、价格可能过期、不构成购买建议(与架构 §7 合规口径一致,不出现「抓取/爬取」等暗示主动爬取的措辞)。
  - **意见反馈**:WeChat 原生 `<button open-type="feedback">`,**非**纠错入口、不发应用请求。

## 影响

- **apps/miniapp**(唯一受影响):新增 `pages/mine` 实页(替换占位)、本地历史读写工具;`pages/compute` 成功后写一条本地历史。复用现有 `pages/compute`、`@unit-price/api-client`、P0 设计 tokens。
- **不触碰合规敏感面**:无服务端用户态、无登录、无抓取、无众包写入;历史仅存本机。
- 无后端 / API / 数据库 / 依赖变更。

## 非目标

- 不做登录 / 微信授权(openid/unionid)——只读 + 本地态不需要。
- 不做收藏 / 关注商品或品类——榜单本身在变,价值打折,等用户真要再加。
- 不做贡献 / 纠错 / 扫码 / 拍照 / 写库(P6 边界)。
- 不做服务端用户态 / 历史云同步——历史纯本地、卸载即失。
- 不做会员 / 积分 / 任何账户体系。
